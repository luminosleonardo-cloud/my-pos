# 🤖 AI Agents — ร้านขายของชำ POS

> เอกสารวางแผน AI Agents สำหรับระบบ POS ร้านขายของชำ  
> ใช้งานได้ทุกอุปกรณ์ (มือถือ / แท็บเล็ต / PC)  
> ขับเคลื่อนด้วย Claude API (claude-sonnet-4-6)

---

## สถาปัตยกรรมภาพรวม

```
Browser (index / inventory / sales)
        │
        ▼
  js/agents.js  ← ตัวกลางเรียก agents ทุกตัว
        │
        ├── Gemini API  (product lookup — ใช้งานได้แล้ว)
        └── Claude API  (agents ทุกตัวด้านล่าง)
                │
                └── localStorage / DB.js  (data source)
```

## Agent 1 — 📦 Stock Monitor (สต็อกแจ้งเตือนอัจฉริยะ)

**วัตถุประสงค์:** วิเคราะห์ข้อมูลสต็อกและยอดขายย้อนหลัง แล้วแนะนำว่าควรสั่งของอะไร เท่าไร เมื่อไร

**ทริกเกอร์:**
- ผู้ใช้กดปุ่ม "วิเคราะห์สต็อก" ในหน้าคลัง
- อัตโนมัติทุกเช้าเมื่อเปิดแอป (background)

**Input:**
```json
{
  "products": [...],       // สินค้าทั้งหมดพร้อม quantity, lowStockThreshold
  "sales_30d": [...],      // ยอดขาย 30 วันล่าสุด
  "today": "2026-05-10"
}
```

**Output ที่ต้องการ:**
```
🔴 หมดสต็อก (3 รายการ)
  - น้ำตาลทราย 1กก. → สั่งด่วน 20 ถุง (ขายเฉลี่ย 3 ถุง/วัน)
  
🟡 เหลือน้อย (5 รายการ)
  - มาม่ากุ้ง → สั่งเพิ่ม 50 ซอง (สต็อก 4 ซอง, ขาย 8/วัน)
  
💡 แนะนำ: ควรสั่งของภายใน 2 วัน ก่อนสต็อกหมด
```

**System Prompt:**
```
คุณเป็นผู้ช่วยจัดการสต็อกร้านขายของชำ วิเคราะห์ข้อมูลสินค้าและยอดขาย
แล้วแนะนำการสั่งของด้วยภาษาไทย กระชับ เข้าใจง่าย
คำนวณจาก: อัตราการขายเฉลี่ย, สต็อกคงเหลือ, ระยะเวลาที่สต็อกจะหมด
```

**สถานะ:** 🔲 วางแผน

## Agent 2 — 💰 Price Advisor (ที่ปรึกษาตั้งราคา)

**วัตถุประสงค์:** แนะนำราคาขายที่เหมาะสม โดยดูจากต้นทุน กำไรเป้าหมาย และสินค้าใกล้เคียง

**ทริกเกอร์:**
- เมื่อผู้ใช้กรอกราคาต้นทุนในฟอร์มเพิ่มสินค้า (แทน chip suggestion เดิม)
- ปุ่ม "ขอคำแนะนำราคา" ในหน้าแก้ไขสินค้า

**Input:**
```json
{
  "product_name": "น้ำตาลทราย 1กก.",
  "category": "เครื่องปรุง",
  "cost_price": 22.00,
  "similar_products": [...]   // สินค้าหมวดเดียวกันในระบบ
}
```

**Output ที่ต้องการ:**
```
📊 วิเคราะห์ราคาสำหรับ "น้ำตาลทราย 1กก."

ต้นทุน: ฿22.00
─────────────────────
• กำไร 20% → ฿27.50  (แข่งขันได้)
• กำไร 30% → ฿31.50  ✅ แนะนำ — ตลาดมักขาย ฿28-35
• กำไร 40% → ฿36.50  (อาจแพงเกินไป)

💡 สินค้าประเภทนี้ในร้านคุณขายเฉลี่ยกำไร 28%
```

**สถานะ:** 🔲 วางแผน

## Agent 3 — 📊 Sales Analyst (นักวิเคราะห์ยอดขาย)

**ทริกเกอร์:**
- ปุ่ม "วิเคราะห์ด้วย AI" ในหน้าประวัติการขาย
- ปุ่ม "สรุปประจำวัน" เมื่อปิดกะ

```json

  "products": [...],
  "compare_prev": true
}
```

```

รายได้รวม: ฿12,450  (+18% จากสัปดาห์ก่อน) 🟢
จำนวนบิล: 234 บิล   เฉลี่ย ฿53/บิล

🏆 สินค้าขายดี 3 อันดับ:
  1. มาม่ากุ้ง — 148 ซอง (฿888)
  2. น้ำดื่มตรา X — 90 ขวด (฿450)
  3. น้ำตาลทราย — 34 ถุง (฿850)

📉 สินค้าขายน้อย:
  - ซอสหอยนางรม — 0 ขวด ใน 7 วัน → พิจารณาลดราคา

⏰ ชั่วโมงขายดี: 07:00–09:00 และ 17:00–19:00
💡 วันอังคารยอดขายต่ำกว่าเฉลี่ย 35% — ลองจัดโปรโมชั่น
```

## Agent 4 — 🏷️ Promotion Builder (สร้างโปรโมชั่น)

**วัตถุประสงค์:** แนะนำและสร้างโปรโมชั่นอัตโนมัติจากข้อมูลสต็อกและยอดขาย

**ทริกเกอร์:**
- ปุ่ม "แนะนำโปรโมชั่น" ในหน้าคลัง
- เมื่อพบสินค้าไม่ขาย > 14 วัน

**Scenarios ที่รองรับ:**
```
📦 สต็อกมาก + ขายช้า   → แนะนำลดราคา X%
🎁 สินค้าคู่กัน         → แนะนำ Bundle 2 ชิ้นราคาพิเศษ
⏰ วันหมดอายุใกล้        → ลดราคาฉุกเฉิน
📅 วันอ่อนแอ             → โปรโมชั่นเฉพาะวัน
```

**Output:** ข้อความโปรโมชั่นพร้อมส่วนลดที่แนะนำ + เหตุผล

**สถานะ:** 🔲 วางแผน

## Agent 5 — 🧾 Receipt Assistant (ผู้ช่วยออกใบเสร็จ)

**วัตถุประสงค์:** สร้างข้อความสรุปการขาย / ส่ง LINE / export รายงาน

**ทริกเกอร์:**
- หลังชำระเงินสำเร็จ (optional: ส่ง LINE)
- ปุ่ม "สรุปวันนี้" ในหน้าขาย

**Features:**
- สรุปยอดขายวันนี้เป็นข้อความ copy ได้
- Format สำหรับส่ง LINE กลุ่มร้าน
- สร้าง PDF รายงาน (future)

**Output ตัวอย่าง:**
```
📋 สรุปยอดขาย 10 พ.ค. 2569
────────────────
บิลทั้งหมด:    47 บิล
ยอดขายรวม:  ฿3,240.00
เงินสด:      ฿2,890.00
QR Pay:       ฿350.00
────────────────
สินค้าที่ขายดี: มาม่า, น้ำดื่ม, ขนม
⚠️ สินค้าหมด 2 รายการ: น้ำตาล, ไข่
```

## Agent 7 — 🎙️ Voice Input (รับคำสั่งด้วยเสียง)

**วัตถุประสงค์:** เพิ่มสินค้าในตะกร้าหรือค้นหาสินค้าด้วยเสียง — มีประโยชน์มากบนมือถือ

**ทริกเกอร์:**
- กดปุ่มไมค์ในหน้าขาย
- พูด: "เพิ่ม มาม่ากุ้ง 3 ซอง"

**Flow:**
```
ผู้ใช้พูด → Web Speech API (transcribe) 
         → Claude parse intent
         → เรียก addToCart() หรือ search()
```

**Intent ที่รองรับ:**
```
"เพิ่ม [ชื่อสินค้า] [จำนวน]"     → addToCart
"ค้นหา [ชื่อสินค้า]"              → searchProduct
"ลบ [ชื่อสินค้า] ออก"             → removeFromCart
"ยอดรวมเท่าไร"                   → readTotal
"ชำระเงิน"                        → openPayment
```

**เทคโนโลยี:** `window.SpeechRecognition` (Chrome/Android รองรับ)

**สถานะ:** 🔲 วางแผน

## Agent 8 — 📷 Smart Scanner (สแกนอัจฉริยะ)

**วัตถุประสงค์:** สแกนบาร์โค้ดหลายชิ้นในภาพเดียว หรือพิมพ์รายการสินค้าจากภาพใบส่งของ

**ทริกเกอร์:**
- ปุ่ม "สแกนใบส่งของ" ในหน้าคลัง
- อัปโหลดรูปใบ invoice

**Flow:**
```
ถ่ายรูปใบส่งของ 
  → Claude Vision อ่านข้อมูล
  → จับคู่กับสินค้าในระบบ
  → แนะนำอัปเดตสต็อก
```

**เทคโนโลยี:** Claude claude-sonnet-4-6 vision (base64 image input)

**สถานะ:** 🔲 วางแผน

## Notes

- Claude API ต้องการ **HTTPS** — GitHub Pages รองรับแล้ว ✅
- API Key เก็บใน localStorage (เหมือน Gemini key เดิม)
- ทุก agent ทำงาน **client-side** ไม่ต้องมี backend
- ข้อมูลทั้งหมดอยู่ใน localStorage — ไม่ส่งข้อมูลลูกค้าออกนอก (privacy ✅)
- Rate limit: Gemini ฟรีมี quota จำกัด, Claude ใช้ Pay-as-you-go
