# แผนการพัฒนาเว็บแอพร้านขายของชำ

## ภาพรวมโปรเจค

เว็บแอพสำหรับร้านขายของชำ รองรับการสแกนบาร์โค้ด จัดการสินค้า และบันทึกการขาย

---

## โครงสร้างไฟล์

```
grocery-app/
├── index.html          ← หน้าหลัก (POS / เลือกสินค้า)
├── inventory.html      ← หน้าคลังสินค้า
├── css/
│   └── style.css       ← สไตล์ทั้งหมด
├── js/
│   ├── app.js          ← logic หน้า POS
│   ├── inventory.js    ← logic หน้าคลัง
│   ├── db.js           ← จัดการ localStorage
│   └── barcode.js      ← ควบคุม barcode scanner
└── assets/
    └── (icons, images)
```

---

## หน้า 1 — POS / เลือกสินค้า (`index.html`)

### ฟีเจอร์
- **สแกนบาร์โค้ด** ผ่านกล้องมือถือหรือ USB barcode scanner
  - ใช้ไลบรารี `QuaggaJS` หรือ `ZXing` สำหรับสแกนผ่านกล้อง
  - รองรับ input จาก USB scanner (keyboard input mode)
- **ค้นหาสินค้า** โดยพิมพ์ชื่อหรือบาร์โค้ด
- **แสดงรายการสินค้า** เป็น grid card พร้อมรูปภาพ ชื่อ ราคา
- **ตะกร้าสินค้า** แสดงรายการที่เลือก ปรับจำนวนได้
- **สรุปยอด** ราคารวม พร้อมปุ่มชำระเงิน
- **แจ้งเตือน** เมื่อสินค้าหมดหรือเหลือน้อย

### UI Layout
```
┌─────────────────────────────────────────┐
│  🛒 ร้านขายของชำ          [คลังสินค้า]  │
├──────────────────────┬──────────────────┤
│ [🔍 ค้นหา / บาร์โค้ด]│   ตะกร้าสินค้า  │
│                      │                  │
│  [สินค้า] [สินค้า]   │  รายการที่เลือก  │
│  [สินค้า] [สินค้า]   │  ─────────────── │
│  [สินค้า] [สินค้า]   │  รวม: 0 บาท     │
│                      │  [ชำระเงิน]      │
└──────────────────────┴──────────────────┘
```

## หน้า 2 — คลังสินค้า (`inventory.html`)

### ฟีเจอร์
- **ตารางรายการสินค้าทั้งหมด** พร้อม sorting และ filtering
- **เพิ่มสินค้าใหม่** (ฟอร์ม: ชื่อ, บาร์โค้ด, ราคา, จำนวน, หมวดหมู่, รูปภาพ)
- **แก้ไขสินค้า** inline หรือ modal
- **ลบสินค้า** พร้อม confirm dialog
- **สแกนบาร์โค้ดเพื่อค้นหา/เพิ่มสินค้า**
- **ค้นหาและกรอง** ตามชื่อ, หมวดหมู่, สต็อกต่ำ
- **นำเข้า/ส่งออก CSV** (optional)

### UI Layout
```
┌──────────────────────────────────────────────┐
│  📦 คลังสินค้า                    [POS หน้า] │
├──────────────────────────────────────────────┤
│  [+ เพิ่มสินค้า]  [🔍 ค้นหา...]  [กรอง ▼]  │
├──────────┬──────────┬───────┬────────┬───────┤
│ ชื่อสินค้า│ บาร์โค้ด │ ราคา  │ จำนวน  │ action│
├──────────┼──────────┼───────┼────────┼───────┤
│ น้ำตาล   │ 8850...  │ 25฿   │ 50     │ ✏️ 🗑️ │
│ ข้าวสาร  │ 8851...  │ 120฿  │ 8 ⚠️   │ ✏️ 🗑️ │
└──────────┴──────────┴───────┴────────┴───────┘
```

## โครงสร้างข้อมูล

### สินค้า (Product)
```json
{
  "id": "uuid",
  "name": "น้ำตาลทราย 1 กก.",
  "barcode": "8850926100022",
  "price": 25.00,
  "quantity": 50,
  "category": "เครื่องปรุง",
  "image": "base64 หรือ URL",
  "lowStockThreshold": 10,
  "createdAt": "2026-05-08T00:00:00Z",
  "updatedAt": "2026-05-08T00:00:00Z"
}
```

### การขาย (Sale)
```json
{
  "id": "uuid",
  "items": [
    { "productId": "uuid", "name": "...", "price": 25, "quantity": 2 }
  ],
  "total": 50.00,
  "createdAt": "2026-05-08T10:30:00Z"
}
```

## เทคโนโลยีที่ใช้

| ส่วน | เทคโนโลยี | เหตุผล |
|------|-----------|--------|
| Frontend | HTML5 + CSS3 + Vanilla JS | ไม่ต้องพึ่ง framework, ใช้งานง่าย |
| Barcode Scanner | ZXing-js / QuaggaJS | รองรับกล้องมือถือและ USB scanner |
| Storage | localStorage (JSON) | ไม่ต้องมี backend, ใช้ได้ทันที |
| Styling | CSS Variables + Flexbox/Grid | responsive รองรับมือถือและ desktop |

---

## ลำดับการพัฒนา

1. **db.js** — ฟังก์ชัน CRUD กับ localStorage
2. **inventory.html + inventory.js** — จัดการสินค้า (ง่ายกว่า, ต้องมีข้อมูลก่อน)
3. **barcode.js** — ระบบสแกนบาร์โค้ด
4. **index.html + app.js** — หน้า POS + ตะกร้า
5. **style.css** — ออกแบบ UI ให้สวยงาม responsive

---

## ข้อควรระวัง

- localStorage มีขีดจำกัด ~5MB → เหมาะสำหรับร้านขนาดเล็ก (~หลายพันรายการ)
- รูปภาพสินค้าควรบีบอัดก่อนเก็บ หรือใช้ URL แทน
- บาร์โค้ดสแกนผ่านกล้องต้องใช้ HTTPS (หรือ localhost)
- รองรับ USB barcode scanner ผ่าน keyboard event โดยอัตโนมัติ
