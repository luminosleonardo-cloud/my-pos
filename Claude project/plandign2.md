# POS Gap Analysis — สิ่งที่ขาดเมื่อเทียบกับ POS เชิงพาณิชย์
> เปรียบกับ Loyverse, Square, Shopify POS, Billme, pos.guru

---

## ฟีเจอร์ปัจจุบัน (สรุป)
- ขายสินค้า / สแกนบาร์โค้ด / ตะกร้า / รับเงินสด / ใบเสร็จ
- คลังสินค้า (เพิ่ม/แก้ไข/ลบ/ค้นหา/สถานะสต็อก)
- ประวัติการขาย (รายวัน/สัปดาห์/เดือน)
- ลิ้นชัก + เครื่องพิมพ์ความร้อน (ESC/POS)
- Gemini AI + OpenFoodFacts (ค้นหาชื่อ/รูปจากบาร์โค้ด)

### 🔴 Priority 1 — จำเป็นสำหรับร้านจริง

| # | ฟีเจอร์ | เหตุผล | ความยาก |
|---|---------|--------|---------|
| 1 | **ส่วนลด (Discount)** | ทุก POS ต้องมี — ลดเป็น % หรือบาท ต่อบิล หรือต่อรายการ | ง่าย |
| 2 | **VAT / ภาษี** | ร้านค้าที่จด VAT ต้องออกใบกำกับภาษี 7% | ปานกลาง |
| 3 | **PromptPay QR** | ไทย — คนซื้อสแกนจ่าย QR ได้เลย ไม่ต้องเงินสดอย่างเดียว | ปานกลาง |

| 5 | **ตั้งค่าร้าน** | ชื่อร้าน, ที่อยู่, เบอร์โทร, โลโก้ — พิมพ์บนใบเสร็จ | ง่าย |
| 6 | **เปิด/ปิดกะ (Shift)** | นับเงินเปิดกะ → ปิดกะดูยอด → พิมพ์สรุปกะ | ปานกลาง |

## ช่องโหว่ที่ต้องแก้ก่อน Code

1. **localStorage มี limit ~5MB** — ถ้าสินค้าเยอะหรือมีรูปภาพ base64 เยอะจะเต็ม → พิจารณา IndexedDB
2. **ไม่มี receipt number** — ใบเสร็จไม่มีเลขที่ ยากต่อการอ้างอิง
3. **ลบ sale แล้วสต็อกไม่คืน** — ถ้าลบประวัติการขาย สต็อกไม่ rollback
4. **ไม่มี confirm ก่อน clear cart** — ปัจจุบันใช้ `confirm()` ซึ่งถูก block บางเบราว์เซอร์

```
Phase 1 (ร้านใช้งานจริงได้)
  ├── ตั้งค่าร้าน (ชื่อ/ที่อยู่/เบอร์)
  ├── ส่วนลดต่อบิล + ต่อรายการ
  ├── PromptPay QR
  └── เปิด/ปิดกะ + สรุปยอดกะ

Phase 2 (ระบบสมบูรณ์)
  ├── VAT 7% + ใบกำกับภาษีอย่างย่อ
  ├── คืนสินค้า / Refund
  ├── Keyboard shortcuts
  └── Export CSV

Phase 3 (ขยายธุรกิจ)
  ├── ระบบพนักงาน + PIN
  ├── Dashboard กราฟ
  ├── Backup JSON
  └── ปรับสต็อก + stock count
```
